1. Общая схема решения такая: программы запускаются одновременно, вторая
программа сразу же засыпает на секунду, пока первая считывает данные и 
записывает их в именованный канал. После того как первая программа записала
текст в канал она засыпает на две секунды. В это время вторая программа забирает
данные из именованного канала, обрабатывает их необходимым образом и записывает
полученный результат в именованный канал, после чего завершается. В конце
первая программа считывает из именованного канала получившиеся данные и записывает
их в выходной файл. После чего также прекращает работу.

2. Действительно разработано консольное приложение, использующее именованные каналы
и соответствующее следующим требоавниям:
- первый процесс читает данные из файла и через именованный канал передает их второму
- второй процесс забирает данные из именованного канала, обрабатывает их и 
возвращает первому через тот же именованный канал
- первый процесс забирает данные из именованного канала и записывает их в 
выходной файл

3. Действительно для задания имен входного и выходного файлов используются
аргументы командной строки

4. Действительно ввод и вывод при работе с файлами осуществляется 
при помощи функций read и write

5. Данные заносятся в буфер за одно считывание и результаты обработки могут быть 
до 5001 байта, что соответствует условиям.

6. Тестовое покрытие лежит в папке tests

7. Для каждого теста есть файл с результатом (каждому тесту input<i>.txt 
соответствует файл output<i>.txt)
