1. Схема решения следующая: первый поток считывает текст
из входного файла и передает его в неименованный канал второму,
а затем засыпает на секунду чтоб второй успел произвести необходимые 
вычисления. Только в этот раз после считывания данных первый поток не 
закрывает для себя этот неименованный канал и не прекращает свою работу,
а лишь засыпает чтобы потом снова проснуться. Пока первый поток спит, 
второй считывает из неименованного канала текст, обрабатывает его и 
записывает обратно в этот же неименованный канал, после чего закрывает 
для себя этот канал с двух сторон и прекращается. Далее первый поток
просыпается, считывает данные из неименованного канала и записывает
получившийся текст в выходной файл. 

2. Действительно разработано консольное приложение, использующее 
неименованные каналы и удовлетворяющее следующим условиям:
- первый процесс читает текст из файла и передают его через 
неименованный канал второму
- второй процесс осуществляет обработку в соответствии с заданием
и возвращает текст первому через тот же неименованный канал
- первый процесс осуществляет вывод текста в выходной файл

3. Действительно для задания имен выходных и выходных файлов 
используются аргументы командной строки

4. Действительно ввод и вывод осуществляются через read и write

5. Файл считывается за один раз, не превышает по объему 5001 байт,
что не противоречит условиям.

6. Тестовое покрытие предоставлено (папка tests)

7. Каждому тестовому файлв input<i>.txt соответствует файл с 
результатом output<i>.txt
