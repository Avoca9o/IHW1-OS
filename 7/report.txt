1. Схема решения похожа на программу на 5 баллов:
Для начала создается именованный канал при помощи команды
mkfifo(name, flags); имя неименованного канала сохранено в переменной name1
и равно fifo1. Первый процесс считывает данные из файла и записывает текст
в fifo1, после чего засыпает на время и ждет пока второй процесс обработает 
текст. Второй же процесс считывает данные из именованного канала fifo1, 
обрабатывает их в соответствии с условием и записывает обратно в fifo1 для 
того чтобы первый процесс мог считать их. После этого второй процесс завершается.
Первый процесс через некоторое время просыпается, считывает данные из fifo1 
и записывает их в выходной файл. После этого работа программы завершается.

2. Действительно разработано консольное приложение, использующее именованные
каналы и действующее по следующей схеме:
- первый процесс читает текст из файла и передает его второму через
именованный канал
- второй процесс получает данные от первого через именованный канал, обрабатывает
их и через этот же именованный канал возвращает первому процессу
- первый процесс осуществляет вывод полученных данных в выходной файл

3. Действительно для задания имен входного и выходного файлов используются 
аргументы командной строки

4. Действительно ввод и вывод при работе с файлами осуществляется через read и write

5. Текст считывает за один раз и не превышает 5001 байт, что не противоречит условиям

6. Тестовое покрытие представлено в папке tests

7. Для каждого тестового файла представлен файл с результатом:
каждому тесту input<i>.txt соответствует выходной файл output<i>.txt
